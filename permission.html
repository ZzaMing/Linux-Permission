1) 파일 속성과 접근 권한

리눅스 파일 시스템 철학

"모든 것은 파일이다(Everything is a file)"라는 철학을 바탕으로, 다중 사용자(Multi-user) 환경에서 파일과 디렉터리 중에서 접근해도 되는 것과 그렇지 않은 것을 구분 하여 접근 권한을 제한할 수 있습니다.

접근 권한

해당 파일을 읽고 쓰고 실행할 수 있는 권한으로 사용자의 파일을 보호하는 가장 기본적인 보안 기능으로, 사용자 카테고리별(User, Group, Others)로 접근 권한을 다르게 부여하여 파일을 보호할 수 있습니다.

권한 UGO 모델

  • User (u): 파일을 소유한 생성자. 가장 강력한 통제력을 가집니다.
  • Group (g): 파일이 속한 논리적 그룹의 구성원들에게 일괄 적용됩니다.
  • Others (o): 소유자도 그룹도 아닌 그 외 모든 시스템 사용자입니다.

파일 속성 분석 문자열

$ ls -l example.sh

유형 (Type) 1자리
  • - 일반 파일
  • d 디렉터리 (폴더)
  • l 심볼릭 링크
권한 (Permission) 9자리
  • r Read (읽기)
  • w Write (쓰기)
  • x Execute (실행)

📔 일반 파일 (File)

📖
읽기 (r)

내부 데이터 및 텍스트의 내용을 단순히 열람할 수 있는 권한

✍
쓰기 (w)

데이터의 내용을 직접적으로 수정하고 덮어써서 저장할 수 있는 권한

⚡
실행 (x)

파일을 메모리에 적재하여 스크립트나 바이너리 프로세스로 실행

📁 디렉터리 (Directory)

🗂
읽기 (r) = 목록 조회

내부에 존재하는 파일과 하위 폴더의 목록을 조회(ls)

📂
쓰기 (w) = 구조 수정

내부에 파일을 생성, 삭제, 이름 변경 등 구조적 수정

🚶
실행 (x) = 탐색 권한

해당 경로로 직접 이동(cd)하고 내부 파일 속성에 접근

💡 주의해야 할 권한 함정

1. 파일 삭제의 종속성

파일 자체에 쓰기(w) 권한이 없더라도, 파일이 위치한 상위 디렉터리에 쓰기(w) 권한이 있다면 해당 파일을 삭제할 수 있습니다. 파일 삭제는 '디렉터리 구조의 수정'으로 간주되기 때문입니다.

2. 실행 권한의 의존성

쉘 스크립트 파일을 실행하려면 운영체제의 프로그램이 파일 내부의 코드를 읽어들여야 처리할 수 있기에 읽기(r) 권한과 실행(x) 권한이 모두 필요합니다.

3. 심볼릭 링크 더미 권한

명령어로 조회 시 심볼릭 링크 파일은 항상 lrwxrwxrwx로 모든 권한이 열려있는 것처럼 표시되지만 이는 껍데기(Dummy) 표기입니다. 실제 접근 통제는 링크가 가리키고 있는 원본 파일의 권한을 100% 따릅니다.

심볼릭 명령어: chmod <대상><연산자><권한> <파일명>

문자와 연산자를 조합하여 파일이나 디렉터리의 권한을 제어하는 방식입니다.
특정 대상의 권한만 직관적으로 추가하거나 제거할 때 매우 유용합니다.

구성 요소

대상 (Who)
  • User (소유자) u
  • Group (그룹) g
  • Others (기타) o
  • All (전체) a
연산자 (Op)
  • 권한 부여 +
  • 권한 회수 -
  • 덮어쓰기 =
권한 (Perm)
  • Read (읽기) r
  • Write (쓰기) w
  • Exec (실행) x

문법 구조 시각화

u
+
Op

+

주요 명령어 예시

$ chmod u+x script.sh
# 소유자(u)에게 실행(x) 권한 부여(+)
$ chmod o-r secret.txt
# 기타 사용자(o)의 읽기(r) 권한 제거(-)
$ chmod a=r notes.txt
# 모든 사용자(a)에게 읽기 권한만 설정(=)
$ chmod go-w shared_file
# 그룹과 기타 사용자의 쓰기 권한 제거

8진수 명령어: chmod <숫자조합> <파일명>

권한(r=4, w=2, x=1)을 합산하여 8진수 숫자로 변환한 뒤,
UGO 전체 권한을 한 번에 일괄 변경할 때 사용합니다.

8진수 권한 조합기

User 7
Group 5
Others 5

명령어 생성 결과

Generated Command
chmod 755 target

권한 분배 차트

최소 권한의 원칙에 따라 사용자(U) 외의 막대가 가급적 낮게 유지되는 것이 안전합니다.

실무 활용 옵션 (Options)

대량의 파일 권한을 효율적으로 관리하거나, 시스템 변경 내역을 안전하게 모니터링하기 위해 사용하는 주요 옵션입니다.

-R (Recursive)

디렉터리와 내부의 모든 하위 파일/폴더 권한을 재귀적으로 일괄 변경합니다.
잘못 사용할 경우 심각한 시스템 장애를 유발할 수 있습니다.

-v (Verbose)

명령어가 적용되는 모든 파일의 처리 상세 내역을 화면에 출력합니다.

-c (Changes)

전체 파일 중 실제로 권한이 변경된 파일의 내역만 필터링하여 출력하므로 모니터링에 유리합니다.

--reference

지정한 원본 파일의 권한을 그대로 복사하여 대상 파일에 동일하게 덮어씁니다.

옵션 적용 예시

$ chmod -R 755 /var/www/html
# 웹 디렉터리 내부의 모든 파일/폴더 권한을 일괄 755로 변경
$ chmod -v 755 app.py
mode of 'app.py' changed from 0644 (rw-r--r--) to 0755 (rwxr-xr-x)
# -v 옵션: 변경 전후의 8진수 및 문자열 권한 상태를 상세히 출력
$ chmod -c u+w *.txt
# 모든 txt 파일에 쓰기 권한을 추가하되 변경된 결과만 출력
$ chmod --reference=app.py test.py
# test.py의 권한을 app.py와 완벽히 동일하게 동기화

⚠️실수 방지 (Danger Zone)

-R옵션으로 시스템 핵심 바이너리의 권한을 변경할 경우 시스템에 치명적이기 때문에 --preserve-root 옵션이 내장되어 있어, 루트 디렉터리에 대한 재귀적 권한 변경을 시스템 레벨에서 차단합니다.

기본 명령어: chown (Change Owner)

파일이나 디렉터리의 실제 소유자(User)를 다른 사용자로 변경할 때 사용합니다. 보안상 최고 관리자(root) 권한이 필요합니다.

chown root file.txt

file.txt의 소유자를 'root'로 변경합니다.

chown user1:admin file.txt

소유자를 'user1'으로, 소속 그룹을 'admin'으로 동시에 변경합니다. 실무에서 가장 널리 쓰이는 형태입니다.

기본 명령어: chgrp (Change Group)

파일이 속한 논리적 그룹만 단독으로 변경할 때 사용합니다. 협업 폴더 관리 시 유용합니다.

chgrp developers src_code/

src_code 디렉터리의 소속 그룹을 'developers'로 변경합니다.

💡 실무 팁 (Tip)

그룹만 변경할 때 chgrp 대신, 콜론을 사용하여 chown :developers src_code/ 처럼 chown 명령어 하나로 통합하여 처리하는 경우가 더 많습니다.

베이스

755 디렉터리 표준 (소유자 전체, 그룹/기타 읽기+탐색)

644 일반 파일 표준 (소유자 읽기+쓰기, 그룹/기타 읽기)

기본 명령어: umask (User file-creation Mode Mask)

새 파일이나 디렉터리 생성 시 부여되지 않아야 할 권한(마스크)을 설정합니다. (기본 디렉터리 777, 파일 666에서 umask 값을 뺌)

결과 연산
777 - 022 = 755
rwxr-xr-x

명령어 문법: chmod 4000 / chmod u+s 등

기본적인 UGO 권한 앞자리에 4, 2, 1의 숫자를 부여하거나 특정 문자를 사용하여 협업 및 보안 충돌을 제어하는 특수 권한을 설정합니다.

💂

SetUID

4000 (s)

📌
